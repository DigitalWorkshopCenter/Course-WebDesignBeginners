Writing Tests

Overview

Having learned about tests in programming, for this activity we will write our first tests to test our code.

Instructions

  • We will start with code created from the create-react-app command. Download Starting Project Here

  • Examine the code. Notice how the test filename matches the component filename that it will test.

  • Let's go ahead an run our first test. Look at package.json and notice the test script. Run the following commands to run your first test:

    • npm install (which you always run when getting started with a new project)

    • npm test (which will run the App.test.js file's tests)

    • After hitting enter you should see something like this:

  • Now, go to App.js, and change Learn React to Learn More. You will see the test rerun automatically, and you will also see the test fail, because it can no longer find the Learn React text in the file.


  • As a developer, I can then change the test to match the Learn More snippet we put in, or I can fix App.js to match the test. You can see how this would be valuable in an application.

  • As a side note, you can quite the test script from running by pressing ctrl + c in your terminal. Then you can always restart it again by executing: npm test.

Now...let's dive a little bit deeper...

  • Create a component called Greeting.js. This component should have a div with a heading and paragraph in it. Render this component in App.js.

  • Delete App.test.js

  • Create Greeting.test.js right next to Greeting.js. Create a testing function in this file. Every test we write should have the following Three A's:

    • Arrange: Set up the test data, test conditions and test environment

    • Act: Run logic that should be tested (e.g. execute function)

    • Assert: Compare execution results with expected results
