What is Testing?

We have been testing our code throughout this entire course. When you write code, run it, find errors and fix them, this is called manual testing. This is absolutely required if we ever want anything to work as programmers. However, there are tools available to us that help make life much easier, make code much more manageable, and make our applications much more robust. This is possible by introducing something called Automated Testing.

Automated Testing

With automated testing, we can write code that will test an entire application automatically. We basically write code that tests your code. You can then test the individual building blocks of your app. While this sounds very technical, it allows you to test all building blocks automatically.

This in conjunction with manual testing is the best solution for stable programming. There are many different types of automated testing. Here are the most common ones:

  • Unit Tests

    • Test the individual building blocks (functions, components) in isolation

    • Projects typically contain dozens or hundreds of unit tests

    • The most common / important kind of test

  • Integration Tests

    • Test the combination of multiple building blocks

    • Projects typically contain a couple of integration tests

    • Also important, but focus on unit tests in most cases

  • End-to-End (e2e) Tests

    • Test complete scenarios in your app as the user would experience them

    • Projects typically contain only a few e2e tests

    • Important but can also be done manually (partially)

What to Test?

  • Test the different building blocks (small, focused test that test a specific thing)

  • Unit Tests: The smallest building blocks that make up your app

  • With these tests in place, we can run them anytime we want or need to, and ensure that every aspect of our application is functioning properly without having to test every small piece.

Where to Write Test Code and How to Test?

  • We need a tool for running our tests and asserting the results

  • We need a tool for "simulating" (rendering) our React app / components.

  • To do this, we will use Jest for running tests and React Testing Library for simulating/rendering components

  • Both of these tools are already created for you when you create a React application using create-react-app
