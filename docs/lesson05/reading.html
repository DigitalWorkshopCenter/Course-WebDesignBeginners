Promises

What is a promise?

From the Mozilla Website, we learn that t fhe Promise object represents the eventual completion (or failure) of an asynchronous operation and its resulting value. A Promise is a proxy for a value not necessarily known when the promise is created. It allows you to associate handlers with an asynchronous action's eventual success value or failure reason. This lets asynchronous methods return values like synchronous methods: instead of immediately returning the final value, the asynchronous method returns a promise to supply the value at some point in the future. W3Schools has a lot of good information on this.

Where to Deploy?

There are a lot of different hosts that you can choose from. Oftentimes you can purchase a domain and hosting from the same website. A lot of hosting providers will let you host for free if you use a domain of theirs. For example, my Professional Portfolio is hosted with netlify for free as a subdomain of the netlify domain. Several companies do something very similar.

You should be aware that not all hosts can host all projects. For example, I once tried to host a Node.js application with GoDaddy, and it was absolutely terrible. They are made to work with PHP and after a lot of pain, the company I was working for ended up having to use a different host. So before you choose a host for your project, make sure that they can support the technologies that you are going to deploy.

The options listed here work with React websites (for the most part). A lot of people like Github Pages because of how closely linked to their Github account it is. But I found that after deploying my React app to Github Pages, there was some funky behavior because Github Pages isn't made to host technologies like React. Netlify, Heroku and Surge (also listed below) are great alternatives, and they have free options as well. And, all three of these can connect to your Github account, and you can set it up so that if you push to your main/master branch on Github, it will do a build and update your production website. It's pretty awesome!

What is a professional portfolio?

A portfolio is a way to document your professional journey and demonstrate your skills and abilities. It can be a collection of your work and accomplishments in either a physical or digital format.

A resume and cover letter are standard requirements for most job applications, but a professional portfolio is something that can help you stand out from other candidates. You can submit a portfolio with your job application, and you can also offer it during a first interview to give employers an in-depth and tangible summary of your best accomplishments.

Why do I need one?

In the Web Development and Software Engineering Industry, having a professional portfolio is of paramount importance. Most job applications online will require to include a link to your github profile, or a professional portfolio because employers want to see what you are capable of and what you've been able to accomplish.

You should think of creating a professional portfolio as an opportunity to showcase yourself, your skills, your accomplishments and your work. You can use it as a portal to access your projects and difficult challenges you've overcome. The portfolio itself is going to be the first showcased project. You should seek to make it look professional, because as they say, you only get one chance to make a first impression. This professional portfolio is your chance.
